# How to Review This Portfolio

## Purpose

This guide helps hiring managers, recruiters, GRC leaders, cybersecurity managers, and consulting clients review the BayouTrust Digital Bank GRC portfolio project.

The repository includes many files, so this guide highlights the best review path based on the reader's interest.

## Quick Review Path

If you only have 5 to 10 minutes, review these files first:

| Step | File | Why It Matters |
|---|---|---|
| 1 | `README.md` | Gives the overall project purpose and navigation. |
| 2 | `repository-summary.md` | Provides a short summary of the full portfolio. |
| 3 | `03-control-assessment/sample-control-results.md` | Shows control assessment and testing ability. |
| 4 | `05-risk-register/enterprise-risk-register.md` | Shows risk identification and scoring ability. |
| 5 | `07-poam-remediation-plan/poam.md` | Shows remediation tracking ability. |
| 6 | `08-executive-reporting/executive-summary.md` | Shows executive communication ability. |

## What This Portfolio Shows

This portfolio shows the ability to:

- Understand business and technology context.
- Scope a cybersecurity GRC assessment.
- Map controls across major frameworks.
- Request and review evidence.
- Assess control effectiveness.
- Identify and score risks.
- Document findings.
- Build remediation plans.
- Track weaknesses through a POA&M.
- Prepare executive-level risk reporting.
- Write policies and procedures.
- Support audit readiness.

## What This Portfolio Is Not

This portfolio is not:

- A real audit
- Legal advice
- Regulatory advice
- A penetration test
- A SOC 2 report
- A PCI DSS report
- A formal NIST assessment
- Based on real client data

## Summary

This guide is intended to make the repository easier to review.

The strongest files for proving practical GRC ability are the control assessment, evidence tracker, risk register, POA&M, executive summary, and audit readiness checklist.
