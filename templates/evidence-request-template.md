# Evidence Request Template

## Purpose

This template is used to request, track, review, and document audit or assessment evidence.

Evidence requests help validate whether controls are designed properly and operating effectively.

## Evidence Status Definitions

| Status | Definition |
|---|---|
| Not Requested | Evidence has not been requested. |
| Requested | Evidence has been requested from the control owner. |
| Received | Evidence has been received but not reviewed. |
| Reviewed | Evidence has been reviewed and documented. |
| Insufficient | Evidence does not fully support the control requirement. |
| Overdue | Evidence was not received by the due date. |

## Evidence Quality Checklist

Before accepting evidence, confirm that it is:

- Complete
- Current
- Relevant to the control
- Approved where required
- Clearly tied to the system or process being tested
- Dated or time-stamped where appropriate
- Retained in the correct evidence repository

## Common Evidence Examples

| Control Area | Example Evidence |
|---|---|
| Access Control | User listings, access review approvals, MFA reports, role matrices |
| Vulnerability Management | Scan reports, remediation tickets, SLA reports |
| Logging and Monitoring | SIEM dashboards, alert review logs, log source inventory |
| Incident Response | IR plan, tabletop exercise records, lessons learned |
| Vendor Risk | Security questionnaires, contracts, vendor risk ratings |
| Data Protection | Encryption settings, data classification standards, DLP reports |
| Backup and Recovery | Backup reports, restoration test results, recovery screenshots |
| Governance | Policies, standards, approval records, committee minutes |

## Summary

This template supports consistent evidence collection and review during GRC assessments, audits, and compliance readiness activities.
