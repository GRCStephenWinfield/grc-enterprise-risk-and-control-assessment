# Repository Summary

## Project Name

GRC Enterprise Risk and Control Assessment

## Fictional Organization

BayouTrust Digital Bank

## Project Purpose

This repository demonstrates a complete Governance, Risk, and Compliance assessment for a fictional banking, fintech, and technology organization.

The project shows how to scope an assessment, map controls across frameworks, request evidence, assess controls, document risks, identify gaps, create a POA&M, develop remediation plans, and report cybersecurity risk to executives.

## Frameworks Demonstrated

This project references the following frameworks and standards:

- NIST Cybersecurity Framework 2.0
- NIST SP 800-53 Rev. 5
- CIS Controls v8.1
- SOC 2 Trust Services Criteria
- PCI DSS v4.0.1
- POA&M methodology

## Main Deliverables

The repository includes the following major deliverables:

| Deliverable | What It Shows |
|---|---|
| Client Scenario | Ability to define business context and risk drivers. |

| Framework Mapping | Ability to connect controls across NIST, CIS, SOC 2, and PCI DSS. |
| Control Assessment | Ability to test controls and document results. |
| Evidence Request Tracker | Ability to request, track, and review audit evidence. |
| Risk Register | Ability to identify, score, assign, and treat enterprise risks. |
| Gap Analysis | Ability to compare current state to target state. |
| POA&M | Ability to track findings, weaknesses, owners, due dates, and validation methods. |
| Executive Reporting | Ability to communicate technical risk in business language. |
| Policies and Procedures | Ability to create governance documentation. |
| Audit Readiness | Ability to prepare for SOC 2, PCI DSS, NIST, and internal audits. |
| Templates | Ability to create reusable GRC tools. |
