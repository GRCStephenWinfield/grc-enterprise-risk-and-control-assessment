# References

## Purpose

This document lists official cybersecurity, compliance, and GRC resources referenced in the BayouTrust Digital Bank portfolio project.

The purpose is to show which frameworks and standards informed the project structure, control mapping, evidence requests, risk register, POA&M, and audit readiness materials.

## Official Framework Resources

| Resource | Description |
|---|---|
| NIST Cybersecurity Framework | High-level cybersecurity risk management framework used to organize governance, identify, protect, detect, respond, and recover outcomes. |
| NIST SP 800-53 | Security and privacy control catalog used for detailed control mapping and assessment. |
| CIS Controls | Prioritized cybersecurity safeguards used to support practical control implementation. |
| SOC 2 Trust Services Criteria | Criteria used to evaluate controls for service organizations. |
| PCI DSS | Payment card security standard used to support payment security readiness. |

## Frameworks Used in This Project

| Framework | Version Used |
|---|---|
| NIST Cybersecurity Framework | 2.0 |
| NIST SP 800-53 | Revision 5 |
| CIS Controls | v8.1 |
| PCI DSS | v4.0.1 |

## Project Use

These references were used to support:

- Framework mapping
- Control assessment
- Evidence request planning
- Risk register development
- Gap analysis
- POA&M tracking
- Audit readiness checklists
- Executive reporting
- Policy and procedure examples

## Important Note

This portfolio does not reproduce full framework text.

Organizations performing real assessments should consult official publications, qualified auditors, legal counsel, compliance professionals, and regulatory guidance as appropriate.

## Summary

The references in this project support a realistic GRC assessment structure and demonstrate awareness of major cybersecurity and compliance frameworks.
