# Framework Version Notes

## Purpose

This document lists the cybersecurity and compliance frameworks referenced in the BayouTrust Digital Bank GRC portfolio project.

The purpose of this file is to clearly identify which framework versions are used in the assessment.

## Framework Versions Used

| Framework | Version Used | Purpose in This Project |
|---|---|---|
| NIST Cybersecurity Framework | 2.0 | Used to organize cybersecurity outcomes across Govern, Identify, Protect, Detect, Respond, and Recover functions. |
| NIST SP 800-53 | Revision 5 | Used for detailed security and privacy control mapping. |
| CIS Controls | v8.1 | Used as a prioritized set of cybersecurity safeguards. |
| SOC 2 Trust Services Criteria | Current Trust Services Criteria | Used to demonstrate readiness for security, availability, confidentiality, processing integrity, and privacy-related controls. |
| PCI DSS | v4.0.1 | Used to demonstrate payment card security readiness. |
| POA&M | Plan of Action and Milestones methodology | Used to track findings, weaknesses, remediation actions, owners, and due dates. |

## Why Version Tracking Matters

Framework version tracking is important because cybersecurity and compliance requirements change over time.

Using the correct framework version helps ensure that:

- Control mapping is accurate.
- Audit evidence aligns with current expectations.
- Findings are tied to the correct requirements.
- Remediation plans address current control language.
- Executive reports are based on current standards.

## Framework Use in This Portfolio

This portfolio does not reproduce the full text of each framework.

Instead, it uses the frameworks to demonstrate practical GRC skills, including:

- Control mapping
- Risk assessment
- Evidence requests
- Control testing
- Gap analysis
- Remediation planning
- Audit readiness
- Executive reporting

## Notes

This is a fictional portfolio project created for demonstration purposes.

The framework references are used for educational and portfolio purposes only. Organizations should consult the official framework publications and qualified compliance professionals when performing real assessments.
