# Technology Environment

## Purpose

This document describes the fictional technology environment for BayouTrust Digital Bank.

The purpose is to establish a realistic technical environment for the GRC assessment, including systems, platforms, data types, vendors, and security tools.

## Environment Overview

BayouTrust Digital Bank operates a hybrid and cloud-based technology environment that supports online banking, mobile banking, payment processing, internal operations, and customer support.

The environment includes cloud infrastructure, corporate systems, identity services, security tools, vendor platforms, and business applications.

## Core Technology Components

| Component | Description |
|---|---|
| Cloud Infrastructure | Hosts banking applications, APIs, databases, and supporting services. |
| Online Banking Platform | Allows customers to access accounts, view balances, and manage transactions. |
| Mobile Banking Application | Provides mobile access to banking and payment services. |
| Payment Processing Platform | Supports ACH activity, debit card transactions, and payment integrations. |
| Identity Provider | Manages employee authentication, MFA, and single sign-on. |
| Customer Database | Stores customer profiles, account information, and related records. |
| SIEM Platform | Collects and monitors security logs from critical systems. |
| Vulnerability Scanner | Identifies infrastructure and application vulnerabilities. |
| Ticketing System | Tracks IT requests, incidents, and remediation work. |
| Backup Platform | Supports backup and recovery for critical systems. |

## Technology Risks

Key technology risks include:

- Unauthorized access to critical systems
- Privileged account misuse
- Unpatched vulnerabilities
- Incomplete logging coverage
- Incomplete asset inventory
- Weak vendor access governance
- Payment environment segmentation gaps
- Insufficient backup restoration testing
- Inconsistent cloud monitoring
