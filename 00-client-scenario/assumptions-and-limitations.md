# Assumptions and Limitations

## Purpose

This document identifies the assumptions and limitations for the BayouTrust Digital Bank GRC portfolio project.

The purpose is to clearly explain the boundaries of the fictional assessment and avoid implying that this project is based on a real client, real audit, or real regulated data.

## Project Assumptions

This portfolio project is based on the following assumptions:

- BayouTrust Digital Bank is a fictional organization.
- The company operates in the banking, fintech, and technology space.
- The company uses cloud infrastructure to support customer-facing and internal systems.
- The company stores, processes, or transmits sensitive customer and payment-related data.
- The company uses third-party vendors to support business and technology operations.
- The organization is preparing for stronger cybersecurity governance and audit readiness.
- The assessment uses common GRC frameworks for demonstration purposes.
- The risks, findings, control results, and evidence examples are fictional.
- The project is designed to demonstrate practical GRC skills.

## Assessment Assumptions

The assessment assumes that:

- Control owners are available to provide evidence.
- Policies and procedures exist for some major control areas.
- Some controls are operating effectively.
- Some controls require improvement.
- Evidence quality varies by control area.
- Management is willing to track remediation through a POA&M.
- The organization wants to improve audit readiness and executive reporting.

## Technology Assumptions

The fictional technology environment assumes the use of:

- Cloud infrastructure
- Online banking application
- Mobile banking application
- Payment processing platform
- Identity provider
- Multi-factor authentication
- SIEM platform
- Vulnerability scanner
- Backup platform
- Ticketing system
- Vendor-managed systems

## Framework Assumptions

This project references the following frameworks and standards:

- NIST Cybersecurity Framework 2.0
- NIST SP 800-53 Rev. 5
- CIS Controls v8.1
- SOC 2 Trust Services Criteria
- PCI DSS v4.0.1
- POA&M methodology

The project does not reproduce the full text of these frameworks. It uses them to demonstrate control mapping, assessment, evidence review, risk management, remediation planning, and reporting.

## Limitations

This project has the following limitations:

- It is not a real audit.
- It is not legal advice.
- It is not regulatory advice.
- It does not include real client data.
- It does not include real customer data.
- It does not include real payment card data.
- It does not include confidential or proprietary information.
- It does not represent an official SOC 2, PCI DSS, NIST, or regulatory assessment.
- It does not include penetration testing.
- It does not include source code review.
- It does not include live system testing.

## Evidence Limitations

The evidence examples in this portfolio are fictional.

Evidence examples are used to demonstrate:

- Evidence request tracking
- Evidence review notes
- Control testing support
- Audit readiness thinking
- Documentation quality expectations

They should not be interpreted as real audit evidence.

## Use of This Portfolio

This portfolio may be used to demonstrate skills in:

- GRC analysis
- Risk assessment
- Control testing
- Evidence review
- Framework mapping
- Gap analysis
- POA&M development
- Policy writing
- Audit readiness
- Executive reporting

## Summary

This project is a fictional GRC portfolio assessment created for professional demonstration purposes.

The assumptions and limitations help define the boundaries of the work and clarify that no real client, customer, payment, or regulated data is used.
