# Out-of-Scope Systems

## Purpose

This document identifies systems, processes, and activities excluded from the BayouTrust Digital Bank GRC assessment.

The purpose is to clearly define what is not included so the assessment remains focused and does not imply coverage of areas that were not reviewed.

## Out-of-Scope Summary

This assessment is focused on Governance, Risk, and Compliance activities. It does not include hands-on technical security testing, legal review, financial audit work, or production system changes.

## Out-of-Scope Items

| Item | Reason Excluded |
|---|---|
| Penetration Testing | This project focuses on GRC assessment, not offensive security testing. |
| Source Code Review | Application code review is not included in this assessment. |
| Physical Security Testing | Physical facilities and physical access testing are not part of this project. |
| Financial Statement Audit | This is not a financial audit or accounting review. |
| Legal Review | Legal interpretation and legal advice are not included. |
| Live Production Testing | No live systems are tested or modified. |
| Social Engineering Testing | Phishing simulations and social engineering tests are excluded. |
| Forensic Investigation | This is not an active incident investigation. |
| Red Team Testing | Adversarial testing is not included. |
| Privacy Impact Assessment | Privacy review is considered only at a high level and not as a formal PIA. |
| Vendor Contract Negotiation | Contract negotiation is outside the scope of this assessment. |
| Regulatory Filing Support | Formal regulatory filings are not included. |

## Out-of-Scope Systems

The following fictional systems are not directly assessed:

| System | Reason Excluded |
|---|---|
| Building Access Control System | Physical access controls are not part of this GRC assessment. |
| Payroll System | Payroll controls are not included in this assessment scope. |
| Marketing Website | The public marketing site is not part of the core banking or payment environment. |
| Development Sandboxes | Development-only environments are excluded unless they connect to production data. |
| Personal Devices | Personal devices are not directly reviewed. |
| Non-Critical Office Tools | General productivity tools are not tested unless they store sensitive assessment evidence. |

## Important Clarification

Although these items are out of scope, they may still represent real business risks in an actual organization.

For this portfolio project, they are excluded to keep the assessment focused on cybersecurity GRC, control testing, evidence review, risk management, POA&M tracking, and executive reporting.

## Limitations

This out-of-scope list means the assessment does not provide conclusions about:

- Technical exploitability of systems
- Legal compliance conclusions
- Financial reporting controls
- Physical security effectiveness
- Source code security
- Employee susceptibility to phishing
- Formal regulatory compliance certification

## Summary

Defining out-of-scope items helps prevent confusion and sets clear expectations.

This assessment focuses on GRC controls, evidence, risk, remediation planning, audit readiness, and leadership reporting.
