# Control Crosswalk Summary

## Purpose

This document summarizes how common cybersecurity control areas map across major Governance, Risk, and Compliance frameworks.

The purpose of this crosswalk is to show how one control area can support multiple cybersecurity, compliance, audit, and risk management requirements.

## Frameworks Mapped

The following frameworks are included in this control crosswalk:

- NIST Cybersecurity Framework 2.0
- NIST SP 800-53 Rev. 5
- CIS Controls v8.1
- SOC 2 Trust Services Criteria
- PCI DSS v4.0.1

## Key Observations

Many cybersecurity controls support more than one framework.

For example, multi-factor authentication supports access control requirements under NIST, CIS Controls, SOC 2, and PCI DSS.

Vulnerability management also supports several frameworks because it helps organizations identify, prioritize, and remediate security weaknesses.

## Summary

This crosswalk helps reduce duplicate work by showing how a single control can support multiple compliance and risk management requirements.

A unified control set allows the organization to manage cybersecurity risk more efficiently and prepare for multiple audits using the same evidence and control documentation.
